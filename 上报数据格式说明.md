# 上报数据格式说明

## 数据结构对比

### 1. 上报图标问题 (type: "icon")

**API端点**: `/admin/vum/uninstall/report_icon`

**数据结构**:
```json
{
  "package": "luci-app-example",
  "comment": "图标显示不正确",
  "type": "icon",
  "timestamp": 1699999999,
  "device_info": {
    "hostname": "OpenWrt",
    "model": "x86_64",
    "system": "Linux"
  }
}
```

### 2. 上报卸载问题 (type: "uninstall")

**API端点**: `/admin/vum/uninstall/report_uninstall`

**数据结构**:
```json
{
  "package": "luci-app-example",
  "comment": "卸载失败，提示依赖错误",
  "type": "uninstall",
  "timestamp": 1699999999,
  "device_info": {
    "hostname": "OpenWrt",
    "model": "x86_64",
    "system": "Linux"
  }
}
```

## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `package` | string | ✅ | 包名，如 `luci-app-passwall` |
| `comment` | string | ❌ | 用户填写的问题描述，可为空 |
| `type` | string | ✅ | **关键字段**：区分问题类型<br>- `"icon"`: 图标问题<br>- `"uninstall"`: 卸载问题 |
| `timestamp` | number | ✅ | Unix时间戳 |
| `device_info` | object | ✅ | 设备信息 |
| `device_info.hostname` | string | ✅ | 主机名 |
| `device_info.model` | string | ✅ | 设备型号/架构 |
| `device_info.system` | string | ✅ | 操作系统类型 |

## 接收端处理逻辑

### PHP 示例 (report.php)

```php
<?php
// 接收 JSON 数据
$json = file_get_contents('php://input');
$data = json_decode($json, true);

// 获取上报类型
$type = $data['type'] ?? 'unknown';
$package = $data['package'] ?? '';
$comment = $data['comment'] ?? '';
$timestamp = $data['timestamp'] ?? time();

// 根据类型分别处理
if ($type === 'icon') {
    // 处理图标问题
    $log_file = '/var/log/icon_reports.log';
    $message = sprintf(
        "[ICON] %s | Package: %s | Comment: %s | Time: %s\n",
        date('Y-m-d H:i:s', $timestamp),
        $package,
        $comment ?: '(无描述)',
        $timestamp
    );
    
} elseif ($type === 'uninstall') {
    // 处理卸载问题
    $log_file = '/var/log/uninstall_reports.log';
    $message = sprintf(
        "[UNINSTALL] %s | Package: %s | Comment: %s | Time: %s\n",
        date('Y-m-d H:i:s', $timestamp),
        $package,
        $comment ?: '(无描述)',
        $timestamp
    );
    
} else {
    // 未知类型
    http_response_code(400);
    echo json_encode(['ok' => false, 'message' => 'Unknown report type']);
    exit;
}

// 写入日志
file_put_contents($log_file, $message, FILE_APPEND);

// 返回成功响应
echo json_encode([
    'ok' => true,
    'success' => true,
    'message' => 'Report received successfully',
    'type' => $type
]);
?>
```

### 数据库存储示例

```sql
CREATE TABLE reports (
    id INT AUTO_INCREMENT PRIMARY KEY,
    type ENUM('icon', 'uninstall') NOT NULL,
    package VARCHAR(255) NOT NULL,
    comment TEXT,
    timestamp INT NOT NULL,
    hostname VARCHAR(255),
    model VARCHAR(100),
    system VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_type (type),
    INDEX idx_package (package),
    INDEX idx_timestamp (timestamp)
);

-- 插入示例
INSERT INTO reports (type, package, comment, timestamp, hostname, model, system)
VALUES ('icon', 'luci-app-example', '图标显示不正确', 1699999999, 'OpenWrt', 'x86_64', 'Linux');
```

## 统计查询示例

### 按类型统计
```sql
SELECT 
    type,
    COUNT(*) as count,
    COUNT(CASE WHEN comment != '' THEN 1 END) as with_comment,
    COUNT(CASE WHEN comment = '' THEN 1 END) as without_comment
FROM reports
GROUP BY type;
```

结果示例：
```
+----------+-------+--------------+------------------+
| type     | count | with_comment | without_comment  |
+----------+-------+--------------+------------------+
| icon     | 150   | 120          | 30               |
| uninstall| 80    | 75           | 5                |
+----------+-------+--------------+------------------+
```

### 按包名统计
```sql
SELECT 
    package,
    SUM(CASE WHEN type = 'icon' THEN 1 ELSE 0 END) as icon_reports,
    SUM(CASE WHEN type = 'uninstall' THEN 1 ELSE 0 END) as uninstall_reports
FROM reports
GROUP BY package
ORDER BY (icon_reports + uninstall_reports) DESC
LIMIT 10;
```

## 关键优势

### ✅ 明确的类型标识
- 即使用户不填写 `comment`，也能通过 `type` 字段准确判断问题类型
- 避免了依赖文本内容分析的不确定性

### ✅ 便于数据分析
- 可以分别统计图标问题和卸载问题的数量
- 可以针对不同类型的问题采取不同的处理策略

### ✅ 扩展性强
- 未来可以轻松添加新的上报类型（如 `type: "bug"`, `type: "feature"`）
- 保持数据结构的一致性

## 注意事项

1. **type 字段是必需的**：接收端应该验证 `type` 字段的存在和有效性
2. **comment 可以为空**：不要依赖 `comment` 来判断问题类型
3. **时间戳统一使用 Unix 时间戳**：便于跨时区处理
4. **设备信息用于统计分析**：可以了解问题在不同设备上的分布情况
