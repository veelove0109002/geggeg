# 上报类型区分说明

## 问题背景
当用户不填写问题描述时，接收端无法判断上报的是图标问题还是卸载问题。

## 解决方案
在两种上报的数据中都添加 `type` 字段，明确标识问题类型。

## 实现细节

### 1. 上报图标问题
**文件**: `luci-app-uninstall/luasrc/controller/uninstall.lua`  
**函数**: `action_report_icon()`  
**修改**: 在 `report_data` 中添加 `type = 'icon'`

```lua
local report_data = {
    package = pkg_name,
    comment = user_comment,
    type = 'icon',  -- 新增：标记为图标问题
    timestamp = os.time(),
    device_info = {...}
}
```

### 2. 上报卸载问题
**文件**: `luci-app-uninstall/luasrc/controller/uninstall.lua`  
**函数**: `action_report_uninstall()`  
**修改**: 在 `report_data` 中添加 `type = 'uninstall'`

```lua
local report_data = {
    package = pkg_name,
    comment = user_comment,
    type = 'uninstall',  -- 新增：标记为卸载问题
    timestamp = os.time(),
    device_info = {...}
}
```

## 关键优势

✅ **即使 comment 为空，也能准确判断问题类型**
```php
// 接收端处理示例
$type = $data['type'];  // 'icon' 或 'uninstall'

if ($type === 'icon') {
    // 处理图标问题
    save_to_icon_reports($data);
} elseif ($type === 'uninstall') {
    // 处理卸载问题
    save_to_uninstall_reports($data);
}
```

✅ **便于数据统计和分析**
```sql
-- 统计各类型问题数量
SELECT type, COUNT(*) as count 
FROM reports 
GROUP BY type;

-- 结果：
-- icon       | 150
-- uninstall  | 80
```

✅ **扩展性强**
未来可以轻松添加新的上报类型：
- `type: "bug"` - 功能bug
- `type: "feature"` - 功能建议
- `type: "performance"` - 性能问题
