# 图标上报功能说明

## 功能概述

为 luci-app-uninstall 添加了图标问题上报功能,用户可以向开发者反馈应用图标显示不正确的问题。

## 功能特点

### 前端功能

1. **上报按钮位置**
   - 在每个应用卡片的图标右下角显示一个红色感叹号按钮
   - 按钮样式:圆形,红色背景,白色感叹号
   - 鼠标悬停时有放大和颜色变化效果

2. **上报对话框**
   - 显示应用图标预览
   - 显示应用中文名称和包名
   - 提供问题描述输入框(可选)
   - 提交和取消按钮

3. **用户体验**
   - 提交时按钮显示"提交中..."状态
   - 提交成功后显示成功通知
   - 提交失败显示错误信息

### 后端功能

1. **接口路径**: `/admin/vum/uninstall/report_icon`

2. **请求参数**
   - `package`: 包名(必填)
   - `comment`: 问题描述(可选)

3. **上报数据内容**
   ```json
   {
     "package": "luci-app-xxx",
     "comment": "用户填写的问题描述",
     "timestamp": 1234567890,
     "device_info": {
       "hostname": "设备主机名",
       "model": "设备型号",
       "system": "系统类型"
     }
   }
   ```

4. **上报目标**
   - 默认上报地址: `https://plugin.vumstar.com/api/report/icon`
   - 使用 POST 方法,JSON 格式
   - 优先使用 curl,回退到 wget 或 uclient-fetch

5. **返回结果**
   ```json
   {
     "ok": true,
     "message": "图标问题已成功上报,感谢您的反馈!",
     "package": "luci-app-xxx"
   }
   ```

## 技术实现

### 前端代码 (main.js)

1. 在 `renderCard` 函数中添加图标容器和上报按钮
2. 新增 `reportIcon` 函数处理上报逻辑
3. 使用模态对话框收集用户反馈

### 后端代码 (uninstall.lua)

1. 新增路由: `action_report_icon`
2. 收集设备信息和用户反馈
3. 构建 JSON 数据并发送到远程服务器
4. 处理多种 HTTP 客户端(curl/wget/uclient-fetch)

## 使用方法

1. 用户在应用列表中发现图标显示不正确
2. 点击该应用图标右下角的红色感叹号按钮
3. (可选)在对话框中填写问题描述
4. 点击"提交上报"按钮
5. 系统自动将信息发送到后台服务器
6. 收到成功或失败的反馈通知

## 注意事项

1. 上报功能需要设备能够访问互联网
2. 上报数据包含设备基本信息(主机名、型号等)
3. 用户可以选择不填写问题描述直接提交
4. 后台服务器地址可根据需要修改

## 后续开发建议

1. 可在后台服务器端开发接收和处理上报数据的 API
2. 可建立图标问题数据库,统计高频问题
3. 可根据上报数据优化图标库
4. 可添加上报历史记录功能
