# 图标上报功能 - 快速诊断指南

## 🚨 问题现象

前端显示"图标问题已成功上报,感谢您的反馈!"，但后端服务器没有收到数据。

## 🔧 快速诊断

### 方法一: 使用自动诊断脚本 (推荐)

```bash
# 在本地 Mac 上运行
cd "/Users/vee/Downloads/kkk 3 7 2 9 3"

# 运行诊断脚本 (默认 OpenWrt IP: 192.168.1.1)
./diagnose_report.sh

# 或指定自定义 IP
./diagnose_report.sh 192.168.2.1
```

脚本会自动检查：
- ✅ 服务器接收端是否正常
- ✅ OpenWrt 设备连接状态
- ✅ 上报接口是否工作
- ✅ 数据是否成功传输
- ✅ 日志和临时文件

### 方法二: 手动检查

#### 1️⃣ 检查服务器接收端

```bash
# 测试服务器是否能接收数据
curl -X POST https://tb.vumstar.com/api/report/icon \
  -H 'Content-Type: application/json' \
  -d '{"package":"luci-app-test","comment":"手动测试"}'

# 预期响应:
# {"ok":true,"success":true,"message":"图标问题已成功上报,感谢您的反馈!","package":"luci-app-test"}
```

**如果返回 404 或 500**:
- 检查 `report.php` 是否已上传到服务器
- 检查 Nginx 配置中的 URL 重写规则
- 查看服务器错误日志

#### 2️⃣ 检查 OpenWrt 日志

```bash
# SSH 登录到 OpenWrt
ssh root@192.168.1.1

# 查看上报相关日志
logread | grep REPORT_ICON

# 应该看到类似这样的日志:
# [REPORT_ICON] package=luci-app-xxx, comment=测试
# [REPORT_ICON] curl_rc=0, http_code=200, response_len=156
# [REPORT_ICON] server_ok=true, response={"ok":true...}
```

**关键指标**:
- `curl_rc=0`: curl 命令执行成功
- `http_code=200`: HTTP 请求成功 (200-299 都是成功)
- `server_ok=true`: 服务器确认接收成功

**常见问题**:
- `http_code=404`: 服务器找不到接口 → 检查 Nginx 配置
- `http_code=500`: 服务器内部错误 → 检查 PHP 错误日志
- `http_code=nil`: 网络连接失败 → 检查网络和防火墙

#### 3️⃣ 检查临时文件

```bash
# 在 OpenWrt 上查看临时文件
ssh root@192.168.1.1

# 查看发送的数据
cat /tmp/icon_report_data.json

# 查看服务器响应
cat /tmp/icon_report_response.txt

# 如果看到 HTTP_CODE:200，说明请求成功
```

#### 4️⃣ 检查服务器数据

```bash
# 查看服务器是否保存了数据
ssh root@tb.vumstar.com

# 查看上报文件
ls -lt /var/www/html/api/report/icon_reports/*.json | head -5

# 查看最新的上报
cat /var/www/html/api/report/icon_reports/*.json | tail -20

# 查看汇总日志
tail -20 /var/www/html/api/report/icon_reports/reports.log
```

## 🎯 问题定位流程图

```
开始
  ↓
服务器能接收 POST 请求吗?
  ├─ 否 → 检查服务器配置 (Nginx + PHP)
  └─ 是 ↓
       OpenWrt 日志显示 http_code=200 吗?
         ├─ 否 → 检查网络连接和 HTTP 状态码
         └─ 是 ↓
              服务器有保存数据吗?
                ├─ 否 → 检查 PHP 文件权限和错误日志
                └─ 是 → ✅ 功能正常!
```

## 📋 常见问题及解决方案

### 问题 1: 服务器返回 404

**原因**: Nginx 配置错误或 PHP 文件路径不对

**解决**:

```bash
# 1. 检查文件是否存在
ssh root@tb.vumstar.com
ls -la /var/www/html/api/report/report.php

# 2. 检查 Nginx 配置
cat /etc/nginx/sites-available/tb.vumstar.com | grep -A 5 "location.*report"

# 3. 测试 Nginx 配置
nginx -t

# 4. 重启 Nginx
systemctl reload nginx
```

### 问题 2: 服务器返回 500

**原因**: PHP 执行错误或权限问题

**解决**:

```bash
# 1. 查看 PHP 错误日志
tail -50 /var/log/php7.4-fpm.log

# 2. 检查目录权限
ls -la /var/www/html/api/report/
ls -la /var/www/html/api/report/icon_reports/

# 3. 修复权限
chmod 644 /var/www/html/api/report/*.php
chmod 777 /var/www/html/api/report/icon_reports/
chown -R www-data:www-data /var/www/html/api/report/
```

### 问题 3: OpenWrt 日志显示 http_code=nil

**原因**: 网络连接失败

**解决**:

```bash
# 在 OpenWrt 上测试网络
ssh root@192.168.1.1

# 1. 测试 DNS 解析
nslookup tb.vumstar.com

# 2. 测试网络连通性
ping -c 3 tb.vumstar.com

# 3. 测试 HTTPS 连接
curl -I https://tb.vumstar.com

# 4. 检查防火墙规则
iptables -L -n | grep -i drop
```

### 问题 4: 日志显示成功但服务器无数据

**原因**: 这是修复前的 bug，需要更新代码

**解决**:

```bash
# 1. 确认已部署修复后的代码
scp luci-app-uninstall/luasrc/controller/uninstall.lua \
    root@192.168.1.1:/usr/lib/lua/luci/controller/

# 2. 清除 LuCI 缓存
ssh root@192.168.1.1 "rm -f /tmp/luci-* && /etc/init.d/uhttpd reload"

# 3. 重新测试
```

## 🔍 详细调试步骤

### 步骤 1: 验证服务器端

```bash
# 直接测试 PHP 文件
curl -X POST https://tb.vumstar.com/api/report/report.php \
  -H 'Content-Type: application/json' \
  -d '{"package":"test","comment":"direct test"}'

# 测试 URL 重写
curl -X POST https://tb.vumstar.com/api/report/icon \
  -H 'Content-Type: application/json' \
  -d '{"package":"test","comment":"rewrite test"}'

# 两个请求应该返回相同的结果
```

### 步骤 2: 验证 OpenWrt 端

```bash
# SSH 到 OpenWrt
ssh root@192.168.1.1

# 手动构造请求
cat > /tmp/test_report.json << 'EOF'
{
  "package": "luci-app-manual-test",
  "comment": "手动测试",
  "timestamp": 1699356789,
  "device_info": {
    "hostname": "OpenWrt",
    "model": "test",
    "system": "Linux"
  }
}
EOF

# 使用 curl 发送
curl -w '\nHTTP_CODE:%{http_code}\n' \
  -X POST \
  -H 'Content-Type: application/json' \
  -d @/tmp/test_report.json \
  'https://tb.vumstar.com/api/report/icon'

# 检查 HTTP 状态码
# 200-299 = 成功
# 404 = 找不到接口
# 500 = 服务器错误
```

### 步骤 3: 端到端测试

```bash
# 1. 在浏览器中访问
http://192.168.1.1/cgi-bin/luci/admin/vum/uninstall

# 2. 打开开发者工具 (F12)

# 3. 点击任意应用的上报按钮

# 4. 在 Network 标签中查看 report_icon 请求:
#    - Request Headers
#    - Request Payload
#    - Response

# 5. 在 Console 标签中查看是否有错误
```

## 🆘 仍然无法解决？

如果按照上述步骤仍然无法解决问题，请收集以下信息：

```bash
# 1. OpenWrt 日志
ssh root@192.168.1.1 "logread | grep REPORT_ICON" > openwrt_logs.txt

# 2. OpenWrt 临时文件
ssh root@192.168.1.1 "cat /tmp/icon_report_data.json" > request_data.txt
ssh root@192.168.1.1 "cat /tmp/icon_report_response.txt" > response_data.txt

# 3. 服务器日志
ssh root@tb.vumstar.com "tail -100 /var/log/nginx/error.log" > nginx_error.txt
ssh root@tb.vumstar.com "tail -100 /var/log/php7.4-fpm.log" > php_error.txt

# 4. 网络测试
ssh root@192.168.1.1 "curl -v https://tb.vumstar.com/api/report/icon" > network_test.txt 2>&1
```

将这些文件发送给开发者进行进一步分析。
