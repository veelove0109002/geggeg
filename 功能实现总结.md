# 图标上报功能实现总结

## 📋 功能概述

为 `luci-app-uninstall` 添加了用户上报图标问题的功能。用户可以通过点击应用图标上的按钮,向开发者反馈图标显示不正确的问题。

## ✅ 已完成的工作

### 1. 后端实现 (`luasrc/controller/uninstall.lua`)

#### 新增路由
```lua
entry({ 'admin', 'vum', 'uninstall', 'report_icon' }, call('action_report_icon'))
```

#### 新增函数 `action_report_icon()`
- 接收参数: `package` (包名), `comment` (问题描述)
- 收集设备信息: 主机名、型号、系统类型
- 构建 JSON 数据并发送到远程服务器
- 支持多种 HTTP 客户端: curl → wget → uclient-fetch
- 返回成功/失败状态

**上报目标地址**: `https://tb.vumstar.com/api/report/icon`

### 2. 前端实现 (`htdocs/luci-static/resources/view/uninstall/main.js`)

#### 修改 `renderCard()` 函数
- 在应用图标上添加红色圆形上报按钮(右下角)
- 按钮样式: 红色背景 + 白色感叹号
- 支持鼠标悬停效果(放大 + 颜色变化)

#### 新增 `reportIcon()` 函数
- 显示上报对话框
- 展示应用图标预览和名称
- 提供可选的问题描述输入框
- 发送上报请求到后端
- 显示成功/失败通知

## 📡 数据流程

```
用户操作
   ↓
点击图标上报按钮
   ↓
填写问题描述(可选)
   ↓
点击提交
   ↓
前端发送 POST 请求
   ↓ /admin/vum/uninstall/report_icon
后端 Lua 控制器
   ↓
收集设备信息
   ↓
构建 JSON 数据
   ↓
发送到远程服务器
   ↓ https://tb.vumstar.com/api/report/icon
后台服务器(需要实现)
   ↓
保存数据 / 发送通知
   ↓
返回结果
   ↓
前端显示通知(成功/失败)
```

## 🔧 技术细节

### 前端技术
- **框架**: LuCI View 扩展
- **UI 库**: 原生 DOM 操作 (E 函数)
- **HTTP 请求**: `_httpJson()` 方法
- **通知系统**: `ui.addNotification()`

### 后端技术
- **语言**: Lua
- **框架**: LuCI Controller
- **HTTP 客户端**: curl / wget / uclient-fetch (自动回退)
- **数据格式**: JSON
- **临时文件**: `/tmp/icon_report_data.json`

### 安全性
- 支持 CSRF Token 验证
- 仅上报应用包名和用户评论
- 设备信息仅包含型号等基本信息(不包含敏感数据)

## 📝 配置说明

### 修改上报地址

如需更改上报目标服务器,编辑 `uninstall.lua`:

```lua
-- 第 91 行左右
local report_url = 'https://tb.vumstar.com/api/report/icon'
-- 修改为你的服务器地址
local report_url = 'https://your-domain.com/api/report/icon'
```

### 自定义按钮样式

编辑 `main.js` 中的 `reportIconBtn` 样式:

```javascript
// 第 508 行左右
'style': 'position:absolute; bottom:0; right:0; width:18px; height:18px; ...'
```

## 🚀 部署步骤

### 方法 1: 手动复制

```bash
# 复制到设备
scp luasrc/controller/uninstall.lua root@192.168.1.1:/usr/lib/lua/luci/controller/
scp htdocs/luci-static/resources/view/uninstall/main.js root@192.168.1.1:/www/luci-static/resources/view/uninstall/

# 清除缓存并重启
ssh root@192.168.1.1 "rm -f /tmp/luci-* && /etc/init.d/uhttpd reload"
```

### 方法 2: 重新编译

```bash
# 进入 OpenWrt 编译环境
cd openwrt
make package/luci-app-uninstall/compile
make package/luci-app-uninstall/install
```

## ⚠️ 已知限制

1. **网络依赖**: 需要设备能访问互联网和目标服务器
2. **后台服务**: 需要自行实现后台接收 API
3. **HTTP 客户端**: 需要设备安装 curl/wget/uclient-fetch 其中之一
4. **无历史记录**: 当前版本不保存用户上报历史
