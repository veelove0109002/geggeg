# 批量卸载功能说明

## 新增功能

已成功为 `luci-app-uninstall` 项目添加批量卸载功能，用户可以多选软件包后进行批量卸载。

## 功能特性

### 1. 批量操作工具栏
- **位置**：搜索框下方
- **样式**：蓝色渐变背景，现代化设计
- **包含元素**：
  - 全选复选框
  - 已选数量显示
  - 批量卸载按钮

### 2. 软件包选择
- **复选框位置**：每个软件包卡片最左侧
- **选择状态**：实时保存每个包的卸载选项（删除配置、卸载依赖、清空缓存）
- **视觉反馈**：选中后复选框显示勾选状态

### 3. 全选功能
- **全选**：点击工具栏的全选复选框，选中当前显示的所有软件包
- **取消全选**：再次点击取消所有选择
- **半选状态**：部分选中时显示半选状态（indeterminate）

### 4. 批量卸载流程

#### 步骤1：选择软件包
- 勾选要卸载的软件包
- 已选数量实时显示在工具栏

#### 步骤2：点击批量卸载
- 批量卸载按钮在有选择时启用
- 点击后弹出确认对话框

#### 步骤3：确认卸载
- 显示所有待卸载的软件包列表
- 显示总数量和警告信息
- 用户可以取消或确认

#### 步骤4：执行卸载
- 显示进度弹窗
- 实时进度条显示整体进度
- 状态文本显示当前正在卸载的包
- 日志默认折叠，可展开查看详细信息

#### 步骤5：完成
- 显示成功和失败的数量
- 点击完成按钮刷新页面
- 自动清空选择状态

## 技术实现

### 状态管理
- `selectedPackages`: 对象，存储选中的软件包及其卸载选项
- `selectAllState`: 布尔值，全选状态

### 核心函数
1. **updateBatchUI()**: 更新批量操作UI状态
   - 更新已选数量显示
   - 启用/禁用批量卸载按钮
   - 更新全选复选框状态（包括半选）

2. **batchUninstall()**: 批量卸载主函数
   - 显示确认对话框
   - 创建进度弹窗
   - 递归卸载每个软件包
   - 实时更新进度和日志

### 事件处理
- 复选框change事件：更新选择状态
- 全选复选框change事件：全选/取消全选
- 批量卸载按钮click事件：触发批量卸载

### 卸载选项
每个选中的软件包会保存以下选项：
- `name`: 包名
- `version`: 版本号
- `purge`: 是否删除配置文件
- `deps`: 是否卸载相关依赖
- `cache`: 是否清空插件缓存

## 用户体验优化

1. **实时反馈**
   - 选择数量实时更新
   - 按钮状态实时变化
   - 进度条平滑过渡

2. **安全确认**
   - 批量卸载前显示确认对话框
   - 列出所有待卸载的包
   - 红色警告提示

3. **详细日志**
   - 日志默认折叠，减少视觉干扰
   - 可展开查看详细执行过程
   - 成功/失败状态清晰标识

4. **错误处理**
   - 单个包失败不影响其他包
   - 最终显示成功和失败统计
   - 详细错误信息记录在日志中

## 使用示例

### 场景1：卸载多个测试插件
1. 勾选3个测试插件
2. 点击"批量卸载"
3. 确认卸载
4. 等待进度完成
5. 查看结果：成功3个，失败0个

### 场景2：使用全选功能
1. 点击"全选"复选框
2. 所有显示的包被选中
3. 取消不需要卸载的包
4. 点击"批量卸载"
5. 确认并执行

### 场景3：搜索后批量卸载
1. 在搜索框输入关键词
2. 筛选出相关软件包
3. 全选或部分选择
4. 批量卸载选中的包

## 注意事项

1. **不可撤销**：批量卸载操作不可撤销，请谨慎操作
2. **依赖关系**：如果选择"卸载相关依赖"，可能影响其他软件包
3. **配置文件**：如果选择"删除配置文件"，配置将永久丢失
4. **系统包**：建议不要批量卸载系统默认插件
5. **网络连接**：批量卸载过程中请保持网络连接稳定
