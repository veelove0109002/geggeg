# 接收端完整部署包 - 使用说明

## 📦 部署包内容

在 `preview/` 目录下已为你准备好3个文件:

```
preview/
├── report.php         # 接收上报的 API 端点
├── list_reports.php   # 获取上报列表的 API
└── view.html          # 可视化管理后台界面
```

## 🚀 部署步骤 (超简单)

### 1️⃣ 上传文件到服务器

```bash
# 方式一: 使用 scp
scp preview/*.php preview/view.html root@tb.vumstar.com:/var/www/html/api/report/

# 方式二: 使用 FTP/SFTP 工具
# 上传 3 个文件到服务器的 /var/www/html/api/report/ 目录
```

### 2️⃣ 设置文件权限

```bash
ssh root@tb.vumstar.com

# 进入目录
cd /var/www/html/api/report/

# 设置权限
chmod 644 *.php *.html
chmod 755 .
chown -R www-data:www-data .
```

### 3️⃣ 配置 Nginx (重要)

编辑 Nginx 配置文件:

```bash
nano /etc/nginx/sites-available/plugin.vumstar.com
```

添加以下配置:

```nginx
server {
    listen 80;
    server_name tb.vumstar.com;
    
    root /var/www/html;
    index index.html index.php;
    
    # 上报接口 - 重定向到 report.php
    location = /api/report/icon {
        rewrite ^ /api/report/report.php last;
    }
    
    # PHP 支持
    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
    
    # 静态文件
    location ~ \.(html|css|js)$ {
        expires 1h;
    }
}
```

重启 Nginx:

```bash
nginx -t  # 测试配置
systemctl reload nginx  # 重载配置
```

### 4️⃣ 测试接收端

```bash
# 测试上报接口
curl -X POST https://tb.vumstar.com/api/report/icon \
  -H 'Content-Type: application/json' \
  -d '{"package":"luci-app-test","comment":"测试上报功能"}'

# 应该返回:
# {"ok":true,"success":true,"message":"图标问题已成功上报,感谢您的反馈!","package":"luci-app-test"}
```

### 5️⃣ 访问管理后台

在浏览器打开:

```
https://plugin.vumstar.com/api/report/view.html
```

你将看到一个漂亮的管理界面,实时显示所有上报记录! 🎉

## 🔧 高级配置

### 修改存储位置

编辑 `report.php` 的第 13 行:

```php
$dataDir = '/path/to/your/custom/directory';
```

### 添加邮件通知

在 `report.php` 的第 79 行后添加:

```php
// 发送邮件通知
mail(
    'admin@vumstar.com',
    "新的图标上报: {$package}",
    "包名: {$package}\n描述: {$comment}\nIP: {$_SERVER['REMOTE_ADDR']}\n时间: " . date('Y-m-d H:i:s'),
    'From: noreply@vumstar.com'
);
```

### 限制上报频率 (防刷)

在 `report.php` 的第 49 行后添加:

```php
// IP 限流检查
$rateLimitFile = $dataDir . '/rate_limit_' . md5($_SERVER['REMOTE_ADDR']) . '.txt';
if (file_exists($rateLimitFile)) {
    $lastTime = (int)file_get_contents($rateLimitFile);
    if (time() - $lastTime < 60) { // 60秒内只能上报一次
        http_response_code(429);
        die(json_encode(['ok' => false, 'message' => '上报过于频繁,请稍后再试']));
    }
}
file_put_contents($rateLimitFile, time());
```

## 🐛 故障排查

### 1. 404 Not Found

**原因**: Nginx 配置错误或文件路径不对

**解决**:
```bash
# 检查文件是否存在
ls -la /var/www/html/api/report/

# 检查 Nginx 配置
nginx -t

# 查看 Nginx 错误日志
tail -f /var/log/nginx/error.log
```

### 2. 500 Internal Server Error

**原因**: PHP 权限问题或语法错误

**解决**:
```bash
# 检查 PHP 错误日志
tail -f /var/log/php7.4-fpm.log

# 确保目录可写
chmod 755 /var/www/html/api/report/
chmod 777 /var/www/html/api/report/icon_reports/
```

### 3. 无法写入文件

**原因**: 目录权限不足

**解决**:
```bash
# 修改所有者
chown -R www-data:www-data /var/www/html/api/report/

# 修改权限
chmod 755 /var/www/html/api/report/
chmod 777 /var/www/html/api/report/icon_reports/
```

### 4. 管理后台显示空白

**原因**: list_reports.php 无法访问

**解决**:
```bash
# 测试 API
curl https://tb.vumstar.com/api/report/list_reports.php

# 检查浏览器控制台
# 按 F12 查看 Network 和 Console 标签
```
