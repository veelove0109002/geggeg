# 图标上报问题 - 解决方案总结

## 📌 问题描述

**现象**: 用户点击上报按钮后，前端显示"图标问题已成功上报,感谢您的反馈!"，但后端服务器没有收到任何数据。

**影响**: 用户以为上报成功，但实际上数据没有被记录，导致开发者无法收到反馈。

## 🔍 根本原因

原代码在 `luci-app-uninstall/luasrc/controller/uninstall.lua` 中存在逻辑缺陷：

```lua
-- ❌ 错误的判断逻辑
local rc = sys.call(curl_cmd)
if rc == 0 then
    success = true  -- 只检查命令是否执行成功
end
```

**问题分析**:
1. `sys.call()` 返回值为 0 只表示 **curl 命令执行成功**
2. 即使服务器返回 404、500 等错误，curl 仍然返回 0
3. 没有检查 HTTP 状态码，无法判断请求是否真正成功
4. 缺少服务器响应验证

## ✅ 解决方案

### 1. 修复 Lua 代码

已修改 `uninstall.lua` 文件，增加以下功能：

#### ✨ 新增 HTTP 状态码检查

```lua
-- ✅ 正确的判断逻辑
local curl_cmd = string.format(
    "curl -w '\\nHTTP_CODE:%%{http_code}' -X POST -H 'Content-Type: application/json' -d @%q '%s' 2>&1",
    tmpfile, report_url
)

-- 提取 HTTP 状态码
local http_code = response:match('HTTP_CODE:(%d+)')

-- 判断成功: curl返回码为0 且 HTTP状态码为 2xx
if rc == 0 and http_code and tonumber(http_code) >= 200 and tonumber(http_code) < 300 then
    success = true
end
```

#### 📝 新增详细日志

```lua
-- 记录请求详情
sys.exec("logger -t luci-app-uninstall '[REPORT_ICON] curl_rc=" .. rc .. ", http_code=" .. http_code .. "'")

-- 记录服务器响应
sys.exec("logger -t luci-app-uninstall '[REPORT_ICON] response=" .. response:sub(1, 200) .. "'")
```

#### ✔️ 新增服务器响应验证

```lua
-- 解析服务器返回的 JSON，确认服务器确实处理成功
local ok_result, server_data = pcall(json.parse, response)
if ok_result and server_data and type(server_data) == 'table' then
    server_ok = server_data.ok or server_data.success
end
```

#### 🐛 保留临时文件用于调试

```lua
-- 注释掉自动删除，方便调试
-- sys.call('rm -f ' .. tmpfile .. ' /tmp/icon_report_response.txt')
```

### 2. 部署步骤

#### 步骤 1: 更新 OpenWrt 设备

```bash
# 复制修复后的文件
scp luci-app-uninstall/luasrc/controller/uninstall.lua \
    root@192.168.1.1:/usr/lib/lua/luci/controller/

# 清除 LuCI 缓存
ssh root@192.168.1.1 "rm -f /tmp/luci-* && /etc/init.d/uhttpd reload"
```

#### 步骤 2: 部署后端服务 (如果还没部署)

```bash
# 上传 PHP 文件
scp preview/report.php root@tb.vumstar.com:/var/www/html/api/report/
scp preview/list_reports.php root@tb.vumstar.com:/var/www/html/api/report/
scp preview/view.html root@tb.vumstar.com:/var/www/html/api/report/

# 设置权限
ssh root@tb.vumstar.com << 'EOF'
cd /var/www/html/api/report/
chmod 644 *.php *.html
mkdir -p icon_reports
chmod 777 icon_reports
chown -R www-data:www-data .
EOF
```

#### 步骤 3: 配置 Nginx

编辑 `/etc/nginx/sites-available/tb.vumstar.com`:

```nginx
server {
    listen 80;
    server_name tb.vumstar.com;
    
    root /var/www/html;
    
    # 上报接口 - 重定向到 report.php
    location = /api/report/icon {
        rewrite ^ /api/report/report.php last;
    }
    
    # PHP 支持
    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
}
```

重启 Nginx:

```bash
nginx -t && systemctl reload nginx
```
