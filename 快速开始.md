# 🚀 图标上报功能 - 快速开始

## 5分钟快速部署

### 第 1 步: 上传文件到设备

假设你的设备 IP 是 `192.168.1.1`:

```bash
# 进入项目目录
cd "kkk 3 7 2 9/luci-app-uninstall"

# 上传 Lua 控制器
scp luasrc/controller/uninstall.lua root@192.168.1.1:/usr/lib/lua/luci/controller/

# 上传前端文件
scp htdocs/luci-static/resources/view/uninstall/main.js \
    root@192.168.1.1:/www/luci-static/resources/view/uninstall/
```

### 第 2 步: 清除缓存

```bash
ssh root@192.168.1.1 "rm -f /tmp/luci-* && /etc/init.d/uhttpd reload"
```

### 第 3 步: 打开浏览器测试

访问: `http://192.168.1.1/cgi-bin/luci/admin/vum/uninstall`

在任意应用图标右下角,你会看到一个红色的圆形按钮 (!)

### 第 4 步: 点击测试

1. 点击红色按钮
2. 在弹出的对话框中可选填写问题描述
3. 点击"提交上报"
4. 查看成功或失败通知

## 🔧 故障排除

### 问题 1: 看不到红色按钮

**原因**: 文件未正确上传或缓存未清除

**解决**:
```bash
# 检查文件是否存在
ssh root@192.168.1.1 "ls -la /www/luci-static/resources/view/uninstall/main.js"

# 强制清除所有缓存
ssh root@192.168.1.1 "rm -rf /tmp/luci-* && /etc/init.d/uhttpd restart"

# 清除浏览器缓存
# 按 Ctrl+Shift+Delete 或 Cmd+Shift+Delete
```

### 问题 2: 点击按钮没反应

**原因**: JavaScript 错误

**解决**:
```bash
# 按 F12 打开浏览器开发者工具
# 查看 Console 标签是否有错误
# 查看 Network 标签确认请求是否发送
```

### 问题 3: 提交后一直显示"提交中..."

**原因**: 后端 API 未实现或网络问题

**解决**:
```bash
# 检查后端文件
ssh root@192.168.1.1 "ls -la /usr/lib/lua/luci/controller/uninstall.lua"

# 查看系统日志
ssh root@192.168.1.1 "logread | grep report"

# 测试后端接口
ssh root@192.168.1.1
curl -X POST 'http://127.0.0.1/cgi-bin/luci/admin/vum/uninstall/report_icon' \
  -d 'package=test&comment=test'
```

### 问题 4: 显示"上报失败,请检查网络连接"

**原因**: 
1. 设备无法访问互联网
2. 后台服务器未实现接收 API
3. 防火墙阻止了请求

**解决**:
```bash
# 测试网络连接
ssh root@192.168.1.1
ping -c 3 tb.vumstar.com

# 测试 HTTP 客户端
wget --spider https://tb.vumstar.com/api/report/icon
curl -I https://tb.vumstar.com/api/report/icon

# 临时方案: 修改上报地址为本地测试服务器
# 编辑 /usr/lib/lua/luci/controller/uninstall.lua
# 将 report_url 改为: http://your-server.com/api/report
```
