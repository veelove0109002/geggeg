# 上报卸载功能说明

## 功能概述
在原有"上报图标问题"功能的基础上，新增了"上报卸载问题"功能，允许用户向开发者反馈卸载过程中遇到的问题。

## 修改内容

### 1. 前端修改 (`main.js`)

#### 1.1 添加上报卸载问题按钮
- **位置**: 卡片左下角，在"上报图标按钮"右侧
- **图标**: `xzbc.png` (垃圾桶警告图标)
- **样式**: 
  - 默认: 白色背景，灰色边框
  - 悬停: 橙色背景 (#f59e0b)，带阴影效果
- **位置计算**:
  - VUM插件: `left: 136px` (在VUM标签右侧)
  - 普通插件: `left: 48px` (在上报图标按钮右侧)

#### 1.2 添加 `reportUninstall()` 函数
完整实现了上报卸载问题的交互逻辑，包括：

**弹窗界面**:
- 标题: "上报卸载问题" (黄色警告图标)
- 包信息展示: 图标 + 中文名 + 包名
- 问题描述输入框: 可选填写
- 提交按钮: 橙色 (#f59e0b)

**提交流程**:
1. 点击提交后禁用按钮，显示"提交中..."
2. 发送 GET 请求到 `/admin/vum/uninstall/report_uninstall`
3. 关闭原弹窗
4. 根据结果显示新弹窗:
   - ✅ 成功: 绿色勾 + "上报成功" + 2秒自动关闭
   - ❌ 失败: 红色叉 + 错误信息 + 关闭按钮
   - ❌ 网络错误: 红色叉 + "请检查网络连接" + 关闭按钮

### 2. 后端修改 (`uninstall.lua`)

#### 2.1 添加路由
```lua
e = entry({ 'admin', 'vum', 'uninstall', 'report_uninstall' }, call('action_report_uninstall'))
e.leaf = true
e.acl_depends = { 'luci-app-uninstall' }
```

#### 2.2 添加 `action_report_uninstall()` 函数
实现了完整的后端处理逻辑：

**参数获取** (多种方式兼容):
1. 表单参数 (`http.formvalue`)
2. URL 参数
3. JSON 请求体
4. URL 编码表单数据

**上报数据结构**:
```json
{
  "package": "包名",
  "comment": "用户描述",
  "type": "uninstall",
  "timestamp": 时间戳,
  "device_info": {
    "hostname": "主机名",
    "model": "设备型号",
    "system": "系统类型"
  }
}
```

**发送方式** (按优先级尝试):
1. `curl` - 支持 POST JSON，可获取 HTTP 状态码
2. `wget` - 备用方案
3. `uclient-fetch` - OpenWrt 原生工具

**日志记录**:
- 使用 `logger -t luci-app-uninstall` 记录调试信息
- 记录参数、HTTP状态码、服务器响应等

**响应格式**:
```json
{
  "ok": true/false,
  "message": "提示信息",
  "package": "包名",
  "debug": {
    "http_code": "HTTP状态码",
    "server_response": true/false
  }
}
```

## 与上报图标功能的区别

| 特性 | 上报图标 | 上报卸载 |
|------|---------|---------|
| 图标 | `copy.png` | `xzbc.png` |
| 悬停颜色 | 红色 (#ff6b6b) | 橙色 (#f59e0b) |
| 标题图标 | 黄色警告 | 黄色警告 |
| 提交按钮 | 蓝色 | 橙色 |
| API路径 | `/report_icon` | `/report_uninstall` |
| 数据类型标记 | 无 | `type: "uninstall"` |
| 临时文件 | `icon_report_data.json` | `uninstall_report_data.json` |

## 使用场景

用户可以在以下情况下使用"上报卸载问题"功能：
- 卸载失败
- 卸载后仍有残留文件
- 卸载过程中出现错误
- 卸载后系统异常
- 其他卸载相关问题

## 技术特点

1. **完全复用上报图标的成功模式**: 相同的UI风格、交互逻辑、错误处理
2. **独立的数据通道**: 使用不同的API端点和数据标记，便于后台区分处理
3. **健壮的兼容性**: 支持多种HTTP工具，多种参数传递方式
4. **详细的日志**: 便于调试和问题追踪
5. **用户友好**: 清晰的视觉反馈，自动关闭成功提示

## 测试建议

1. 点击"上报卸载问题"按钮，验证弹窗正常显示
2. 填写问题描述，点击提交，验证提交流程
3. 检查系统日志: `logread | grep luci-app-uninstall`
4. 验证成功/失败/网络错误三种情况的反馈弹窗
5. 确认按钮位置在不同类型插件卡片上正确显示

1. 清理 LuCI 缓存: `rm -f /tmp/luci-*`

3. 刷新浏览器页面 (Ctrl+F5 强制刷新)
