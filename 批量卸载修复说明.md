# 批量卸载功能修复说明

## 问题描述

批量卸载功能在执行时全部失败，显示如下错误：
```
[1/3] 开始卸载: 迅游 (luci-app-xunyou)
✗ 卸载失败: 迅游 (luci-app-xunyou)

[2/3] 开始卸载: DDNSTO路由远程 (luci-app-ddnsto)
✗ 卸载失败: DDNSTO路由远程 (luci-app-ddnsto)

[3/3] 开始卸载: 易有云 (luci-app-linkease)
✗ 卸载失败: 易有云 (luci-app-linkease)
```

## 问题原因

批量卸载逻辑与单个卸载逻辑不一致：

1. **缺少GET请求fallback**：批量卸载只发送POST请求，当POST失败时没有尝试GET请求
2. **缺少详细日志**：没有记录请求和响应的详细信息，难以调试
3. **错误处理不完整**：POST失败后直接标记为失败，没有重试机制

## 修复内容

### 1. 添加GET请求fallback

参考单个卸载的逻辑，在POST请求失败后自动尝试GET请求：

```javascript
}).then(function(res){
    println('< Response: ' + JSON.stringify(res));
    if (res && res.ok) {
        // 成功处理
        println('✓ ' + _('卸载成功: ') + fullName);
        successCount++;
        currentIndex++;
        var progressBarEl = document.getElementById('batch-progress-bar');
        if (progressBarEl) progressBarEl.style.width = ((currentIndex / packages.length) * 100) + '%';
        uninstallNext();
        return;
    }
    // POST失败，尝试GET
    println('! POST 失败或返回非成功，尝试 GET…');
    var getUrl = L.url('admin/vum/uninstall/remove') + '?' +
        (token ? ('token=' + encodeURIComponent(token) + '&') : '') +
        ('package=' + encodeURIComponent(pkg.name) + '&purge=' + (pkg.purge ? '1' : '0') + '&removeDeps=' + (pkg.deps ? '1' : '0') + '&clearCache=' + (pkg.cache ? '1' : '0'));
    println('> GET ' + getUrl);
    return self._httpJson(getUrl, { method: 'GET', headers: { 'Accept': 'application/json' } }).then(function(r2){
        println('< Response: ' + JSON.stringify(r2));
        if (r2 && r2.ok) {
            println('✓ ' + _('卸载成功: ') + fullName);
            successCount++;
        } else {
            println('✗ ' + _('卸载失败: ') + fullName);
            failCount++;
        }
        currentIndex++;
        var progressBarEl = document.getElementById('batch-progress-bar');
        if (progressBarEl) progressBarEl.style.width = ((currentIndex / packages.length) * 100) + '%';
        uninstallNext();
    });
})
```

### 2. 添加详细日志

在请求前后添加日志输出：

```javascript
println('> POST ' + removeUrl);
println('> body: ' + formBody);

self._httpJson(removeUrl, {
    method: 'POST',
    headers: { ... },
    body: formBody
}).then(function(res){
    println('< Response: ' + JSON.stringify(res));
    // ...
})
```

### 3. 改进错误处理

在catch块中使用更清晰的错误信息：

```javascript
}).catch(function(err){
    println('! Error: ' + String(err));
    failCount++;
    currentIndex++;
    var progressBarEl = document.getElementById('batch-progress-bar');
    if (progressBarEl) progressBarEl.style.width = ((currentIndex / packages.length) * 100) + '%';
    uninstallNext();
});
```

## 修复效果

修复后，批量卸载功能将：

1. ✅ **自动重试**：POST失败后自动尝试GET请求
2. ✅ **详细日志**：记录所有请求和响应，便于调试
3. ✅ **更高成功率**：与单个卸载逻辑一致，提高卸载成功率
4. ✅ **更好的用户体验**：用户可以在日志中看到详细的执行过程

## 测试建议

1. 选择2-3个软件包进行批量卸载
2. 展开日志查看详细的执行过程
3. 确认卸载成功后，检查系统中是否已删除相关文件
4. 如果仍然失败，查看日志中的错误信息进行进一步调试

## 技术细节

- **文件位置**：`luci-app-uninstall/htdocs/luci-static/resources/view/uninstall/main.js`
- **修改行数**：约947-992行（批量卸载的HTTP请求部分）
- **修改方式**：完全重写批量卸载的HTTP请求逻辑，使其与单个卸载逻辑保持一致

## 注意事项

1. 修复后需要刷新浏览器缓存（Ctrl+F5或Cmd+Shift+R）
2. 如果问题仍然存在，请检查后端API是否正常工作
