# 快速部署指南

## 📦 完整部署流程

### 第一步：部署接收端（服务器端）

#### 1. 创建 report.php
将以下代码保存为 `report.php` 并上传到服务器：

```bash
# 上传到服务器
scp report.php root@tb.vumstar.com:/var/www/html/
```

#### 2. 设置权限
```bash
ssh root@tb.vumstar.com

# 创建数据目录
mkdir -p /var/www/html/reports
chmod 777 /var/www/html/reports

# 设置文件权限
chmod 644 /var/www/html/report.php
```

#### 3. 测试接收端
```bash
# 测试图标上报
curl -X POST https://tb.vumstar.com/report.php \
  -H "Content-Type: application/json" \
  -d '{"package":"test","comment":"测试","type":"icon","timestamp":1705305022,"device_info":{}}'

# 预期响应：
# {"ok":true,"success":true,"message":"上报成功","type":"icon","package":"test"}
```

### 第二步：部署客户端（OpenWrt 端）

#### 1. 上传修改后的文件
```bash
# 上传 main.js
scp luci-app-uninstall/htdocs/luci-static/resources/view/uninstall/main.js \
  root@192.168.1.1:/usr/lib/lua/luci/view/uninstall/

# 上传 uninstall.lua
scp luci-app-uninstall/luasrc/controller/uninstall.lua \
  root@192.168.1.1:/usr/lib/lua/luci/controller/
```

#### 2. 清理缓存并重载
```bash
ssh root@192.168.1.1

# 清理 LuCI 缓存
rm -f /tmp/luci-*

# 重载 Web 服务
/etc/init.d/uhttpd reload
# 或
/etc/init.d/nginx reload
```

#### 3. 测试功能
1. 打开浏览器访问 OpenWrt 管理界面
2. 进入"高级卸载"页面
3. 找到任意插件卡片
4. 点击左下角的两个按钮：
   - 📋 上报图标问题（红色悬停）
   - 🗑️ 上报卸载问题（橙色悬停）
5. 填写或不填写描述，点击提交
6. 查看是否显示"上报成功"弹窗

### 第三步：验证数据

#### 1. 查看服务器端数据
```bash
ssh root@tb.vumstar.com

# 查看目录结构

# 预期输出：
```

#### 2. 查看客户端日志

## 🔍 问题排查

### 问题1：提交后显示"上报失败"

**可能原因**：
- 网络连接问题
- 服务器端 report.php 不可访问
- 权限问题

**排查步骤**：
```bash
# 1. 测试网络连接
ssh root@192.168.1.1
ping tb.vumstar.com

# 2. 测试 HTTPS 访问
curl -I https://tb.vumstar.com/report.php

# 3. 查看详细日志
logread | grep luci-app-uninstall

# 4. 手动测试上报
curl -X POST https://tb.vumstar.com/report.php \
  -H "Content-Type: application/json" \
  -d '{"package":"test","type":"icon","timestamp":1705305022,"device_info":{}}'
```

### 问题2：服务器端没有收到数据

**可能原因**：
- PHP 没有写入权限
- 目录不存在

**排查步骤**：
```bash
# 1. 检查目录权限
ls -la /var/www/html/reports/

# 2. 检查 PHP 错误日志
tail -f /var/log/nginx/error.log
# 或
tail -f /var/log/apache2/error.log

# 3. 测试 PHP 写入
php -r "file_put_contents('/var/www/html/reports/test.txt', 'test');"
cat /var/www/html/reports/test.txt
```

### 问题3：无法区分图标和卸载问题

**检查步骤**：
```bash
# 1. 查看上报的 JSON 数据
cat /var/www/html/reports/icon/20240115_*.json | grep type
cat /var/www/html/reports/uninstall/20240115_*.json | grep type

# 2. 应该看到：
# "type": "icon"
# "type": "uninstall"

# 3. 如果没有 type 字段，检查客户端代码
ssh root@192.168.1.1
grep "type = 'icon'" /usr/lib/lua/luci/controller/uninstall.lua
grep "type = 'uninstall'" /usr/lib/lua/luci/controller/uninstall.lua
```

## 🔧 维护任务

### 定期清理旧数据
```bash
# 创建清理脚本
cat > /var/www/html/cleanup_reports.sh << 'EOF'
#!/bin/bash
# 删除 30 天前的 JSON 文件
find /var/www/html/reports/icon/ -name "*.json" -mtime +30 -delete
find /var/www/html/reports/uninstall/ -name "*.json" -mtime +30 -delete

# 压缩旧日志
find /var/www/html/reports/ -name "*.log" -mtime +7 -exec gzip {} \;

echo "清理完成: $(date)"
EOF

chmod +x /var/www/html/cleanup_reports.sh

# 添加到 crontab（每天凌晨 2 点执行）
crontab -e
# 添加：
# 0 2 * * * /var/www/html/cleanup_reports.sh >> /var/log/cleanup_reports.log 2>&1
```

### 备份数据
```bash
# 创建备份脚本
cat > /var/www/html/backup_reports.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/var/backups/reports"
DATE=$(date +%Y%m%d)

mkdir -p $BACKUP_DIR
tar -czf $BACKUP_DIR/reports_$DATE.tar.gz /var/www/html/reports/

# 删除 7 天前的备份
find $BACKUP_DIR -name "reports_*.tar.gz" -mtime +7 -delete

echo "备份完成: $BACKUP_DIR/reports_$DATE.tar.gz"
EOF

chmod +x /var/www/html/backup_reports.sh

# 添加到 crontab（每天凌晨 3 点执行）
# 0 3 * * * /var/www/html/backup_reports.sh >> /var/log/backup_reports.log 2>&1
```
