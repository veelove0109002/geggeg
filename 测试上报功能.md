# 测试图标上报功能

## 快速测试步骤

### 1. 部署更新的代码

将修改后的文件部署到 OpenWrt/iStoreOS 设备:

```bash
# 复制 Lua 控制器文件
scp luci-app-uninstall/luasrc/controller/uninstall.lua root@192.168.1.1:/usr/lib/lua/luci/controller/

# 复制 JavaScript 前端文件
scp luci-app-uninstall/htdocs/luci-static/resources/view/uninstall/main.js root@192.168.1.1:/www/luci-static/resources/view/uninstall/

# 清除 LuCI 缓存
ssh root@192.168.1.1 "rm -f /tmp/luci-* && /etc/init.d/uhttpd reload"
```

### 2. 访问页面

打开浏览器访问: `http://192.168.1.1/cgi-bin/luci/admin/vum/uninstall`

### 3. 查找上报按钮

在任意应用卡片的图标右下角,可以看到一个红色圆形按钮,上面有白色感叹号(!)

### 4. 点击上报

点击红色按钮,会弹出上报对话框,包含:
- 应用图标预览
- 应用名称(中文名 + 包名)
- 问题描述输入框
- 提交和取消按钮

### 5. 提交反馈

可选择填写或不填写问题描述,直接点击"提交上报"按钮

### 6. 查看结果

- 成功: 显示绿色通知"图标问题已成功上报,感谢您的反馈!"
- 失败: 显示红色错误通知

## 手动测试 API

### 使用 curl 测试后端接口

```bash
# SSH 登录到设备
ssh root@192.168.1.1

# 测试上报接口
curl -X POST \
  'http://127.0.0.1/cgi-bin/luci/admin/vum/uninstall/report_icon' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -d 'package=luci-app-adguardhome&comment=图标显示不正确'
```

### 预期响应

成功情况:
```json
{
  "ok": true,
  "message": "图标问题已成功上报,感谢您的反馈!",
  "package": "luci-app-adguardhome"
}
```

失败情况(网络问题):
```json
{
  "ok": false,
  "message": "上报失败,请检查网络连接",
  "details": "..."
}
```

## 调试技巧

### 1. 查看浏览器控制台

按 F12 打开开发者工具,查看:
- Network 标签: 检查 API 请求和响应
- Console 标签: 查看 JavaScript 错误

### 2. 查看服务器日志

```bash
# 查看系统日志
logread -f

# 查看 uhttpd 日志
tail -f /var/log/uhttpd.log
```

### 3. 检查临时文件

上报过程中会创建临时文件:
```bash
ls -la /tmp/*icon_report*
cat /tmp/icon_report_data.json
cat /tmp/icon_report_response.txt
```

## 已知限制

1. **网络依赖**: 需要设备能够访问 `tb.vumstar.com`
2. **HTTP 客户端**: 需要 curl、wget 或 uclient-fetch 其中之一
3. **后台服务**: 需要后台服务器实现接收 API (目前可能返回 404)

## 后台服务器开发建议

如需开发后台接收服务,参考以下 API 规范:

### 接收端点
- URL: `https://tb.vumstar.com/api/report/icon`
- 方法: POST
- Content-Type: application/json

### 请求体示例
```json
{
  "package": "luci-app-adguardhome",
  "comment": "图标显示模糊",
  "timestamp": 1704067200,
  "device_info": {
    "hostname": "OpenWrt",
    "model": "aarch64",
    "system": "Linux"
  }
}
```

### 推荐的响应
```json
{
  "success": true,
  "message": "Report received",
  "report_id": "uuid-xxx-xxx"
}
```

## 常见问题

### Q: 点击按钮没有反应?
A: 检查浏览器控制台是否有 JavaScript 错误,确认文件已正确部署

### Q: 提交后一直显示"提交中..."?
A: 检查网络连接,查看后台日志确认请求是否发送成功

### Q: 如何修改上报地址?
A: 编辑 `uninstall.lua` 文件的 `action_report_icon` 函数,修改 `report_url` 变量
